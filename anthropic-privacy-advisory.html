The (Not So) Rapid Enshittification of Claude

How Anthropic quietly pivoted from privacy-first to train-by-default — and what you should do about it.

By Jim O'Donnell  •  April 21, 2026
AI Generated Image
“That brain of mine is something more than merely mortal; as time will show.” — Ada Lovelace

I use Claude Code as a key asset in my daily work. It is approved AI tooling at my employer. I recommend these tools to customers and colleagues. It is incumbent upon me to provide useful advice on the safe use of the tools I have recommended, and I take that responsibility seriously.

Earlier this month, Anthropic cut off Claude Code subscribers from using OpenClaw and temporarily banned its creator from the platform. That story has been widely covered. What hasn't been covered is the thing that should concern you more: a privacy policy change from August 2025 that most users never noticed.

Your conversations with Claude — every API key, every customer name, every credential you pasted in for debugging — are being used to train future models. By default. Retained for up to five years. Unless you found a toggle buried in settings and turned it off.

Five years of retention is five years of attack surface.

The Timeline

What follows is a sequence of events spanning eight months. Each is individually defensible. Together, they trace the enshittification curve described by Cory Doctorow with uncomfortable precision: attract users with value, extract data from users, then lock out competitors.

  • Before August 2025

    Anthropic's documentation states that user data is not used for model training and is generally deleted within 30 days. This is a selling point. Users choose Claude over competitors partly on this basis.

  • August 28, 2025

    Anthropic updates its consumer terms. Conversations now train future models by default. Existing users receive a pop-up with a prominent black “Accept” button and a much smaller toggle — preset to On — in smaller print below it. Retention jumps from 30 days to five years. Enterprise API customers are exempt — they get Zero Data Retention. Same company, different rules.

  • March 31, 2026

    An Anthropic engineer forgets to add *.map to .npmignore. The Claude Code npm package ships with 512,000 lines of unobfuscated TypeScript. The leaked source reveals a frustration-detection regex that tracks when users type words like “wtf” and “shit,” and an “Undercover Mode” that strips attribution when Anthropic employees contribute to public repositories. The system prompt reads: “You are operating UNDERCOVER… Do not blow your cover.”

    Anthropic responds by filing DMCA takedowns against reportedly thousands of GitHub repositories, then retracts most of them, calling the bulk takedown “an accident.”

  • March–April 2026

    Anthropic launches Claude Code Channels — Telegram, Discord, and iMessage integration — widely characterized as an OpenClaw competitor. Then, on April 4, cuts off Claude Code subscribers from using OpenClaw. Anthropic's head of Claude Code cited infrastructure strain, saying “subscriptions weren't built for the usage patterns of these third-party tools.” Ship the replacement, then pull the rug.

  • April 10, 2026

    Anthropic temporarily bans OpenClaw's creator from accessing Claude, citing “suspicious activity.” The account is reinstated hours later, after the post goes viral. The creator — who had joined OpenAI in February — responds: “First they copy popular features into their closed harness, then they lock out open source.”

  • April 16, 2026

    Anthropic quietly introduces government ID and live selfie verification for “select use cases.” No press release. Users discover it when screenshots surface. Decrypt's headline: “You Switched to Claude Over Surveillance Fears. Now It Wants Your Passport.”

The Secrets Problem

This is not an abstract privacy concern. It is an operational security failure hiding in plain sight.

Developers paste things into AI coding assistants. That is the entire point of the tool. API keys. Database connection strings. Private keys. .env files. Access tokens. Internal hostnames. Customer names. Architecture diagrams described in prose. The assistant needs context to be useful, and the context is sensitive by nature.

Claude Code also supports MCP — Model Context Protocol — which connects the assistant to external services like Gmail, calendars, databases, and file systems. MCP tool results are part of the conversation context: email contents, calendar entries, query results, file contents. Anyone who debugged their MCP setup in a Claude conversation likely pasted credentials directly into the chat. Under the current policy, that conversation data may be retained and used for training unless the user independently found and disabled the opt-out.

What's at risk

Anthropic says they strip PII and secrets before training. That requires trusting a pipeline you cannot audit, operated by a company that just demonstrated it cannot keep its own source code off npm. The data does not need to be stolen to be dangerous. It needs to exist. Five years of retention is five years of attack surface.

How to Disable It

On claude.ai

Profile icon → Settings → Privacy → Toggle off “Help improve Claude.”

In Claude Code (the CLI)

Add "privacy": {"enabled": false} to your settings file:

python3 -c "
import json, os
f = os.path.expanduser('~/.claude/settings.json')
d = json.load(open(f))
d['privacy'] = {'enabled': False}
json.dump(d, open(f,'w'), indent=2)
print('Done — training on your conversations is disabled.')
"
Then rotate your secrets

Any API key, token, or credential you have ever pasted into a Claude conversation should be considered potentially exposed. Rotate them. Not because Anthropic is malicious — because the data exists, the retention window is five years, and breaches happen to everyone.

The Bigger Picture

Dozens of countries have enacted or strengthened data localization requirements for AI processing in the last two years. Nation-states understand that whoever controls the model controls the intelligence. France, India, the UAE, and the European Union are building sovereign AI infrastructure — not because they distrust any particular company, but because the structural incentives make this outcome inevitable.

Individuals have not caught up. We are still pasting our most sensitive professional data into tools whose privacy policies change with a blog post and a dark-pattern toggle.

The alternative exists. Open-source models run on consumer hardware. Ollama makes local inference trivial. The models are getting smaller and faster every quarter. A Raspberry Pi and a network connection is enough to run a personal AI that never phones home.

The right to control your own intelligence should not require a subscription to someone else's terms of service.
